# 简历项目经验输出模板

## 格式规范

基于《前端工程师简历规范》中的项目经历描述格式，每个项目应包含：

1. 项目名称 + 在线链接
2. 技术栈（核心 3-5 项）
3. 一句话说明项目是什么
4. 2-3 个技术亮点（STAR 法则 + 量化数据）

## 输出模板

```markdown
# <项目名称>

> <项目时间范围> | <项目角色> | <在线链接>

## 项目简介

<一句话说明项目是做什么的，解决什么问题，面向什么用户。用业务术语，不罗列技术栈。>

## 技术栈

**核心：** <3-5 项>
**辅助：** <2-4 项>

## 核心贡献

### <贡献标题 1>

<STAR 描述的正文段落，2-3 句话。包含：做了什么、为什么、取得什么结果。>
<至少包含 1 个量化数据或具体指标。>

### <贡献标题 2>

<同上，角度与贡献 1 互补>

### <贡献标题 3（可选）>

<同上>

## 项目亮点

- <亮点 1>
- <亮点 2>
- <亮点 3>

## 补充材料

<details>
<summary>技术细节与中间分析结果</summary>

### 完整依赖清单
<从 tech-stack.md 中提取>

### 架构说明
<从 project-overview.md 中提取>

### 提交统计
<从 contributions.md 中提取>

</details>
```

## 质量检查清单

生成后逐项自检：

- [ ] 项目名称准确
- [ ] 在线链接（如有）已包含
- [ ] 技术栈在 8 项以内，核心突出
- [ ] 项目简介用业务语言，非技术堆砌
- [ ] 至少 2 个 STAR 贡献条目
- [ ] 每个条目有至少 1 个量化数据或 `<待补充>`
- [ ] 全文第一人称 "我"
- [ ] 补充材料折叠在 `<details>` 中
- [ ] 无编造数据
- [ ] `<待补充>` 项已在前文汇总列出

## 文件命名

`${start_time}-${end_time}-${project_name}.md`

示例：`2024-03-15-2025-01-10-trial-system.md`
