# 面试准备清单生成工作流

## 概述

解析用户提供的 Markdown 简历，从中提取专业技能、工作经历和项目经历，逐一映射到对应的面试知识点，生成一份按优先级排序的面试准备清单。

输出文件：`<姓名>-面试准备清单.md`，保存到当前工作目录。

## 核心原则

1. **从简历出发**：所有知识点必须有简历中的技能或项目作为锚点，不凭空罗列面试题
2. **项目驱动**：每个技术点都关联用户的真实项目经历，面试回答才能言之有物
3. **分层优先级**：核心技能（深度实践）> 辅助技能（项目经验）> 技术视野
4. **避虚就实**：不列"请描述 React 生命周期"这种纯概念题，而是结合项目问"你在 XX 项目中为什么用 Hooks 而不是 Class？"

## 工作流

### 第一步：获取简历文件

1. **用户在调用时提供了文件路径** → 直接读取
2. **未提供文件路径** → 扫描当前目录下的 `*.md` 文件，筛选看起来像简历的文件（标题含"工程师""简历""CV"等），让用户选择；如只有一个匹配则直接确认
3. **当前目录无匹配** → 询问用户提供路径

确认文件后向用户展示：

```
找到简历文件：<文件名>
姓名：<从 H1 提取>
目标职位：<从 H1 提取>
确认从此文件生成面试准备清单？
```

### 第二步：解析简历

读取 Markdown 文件，提取以下信息：

#### 2.1 专业技能

从 `## 专业技能` 区域解析所有分组和技能项：

```
解析规则：
- 以 **分组名：** 或 **分组名** 开头的行识别为一个分组
- 分组名后的内容按顿号/逗号/斜杠拆分为技能项
- 每个技能项去空白、去多余符号
```

输出解析结果供用户确认：

```
📚 已识别技术栈（共 x 项，y 个分组）

**框架/语言**：React、TypeScript、Electron、Next.js、...
**工程化/工具链**：GitHub Actions、CI/CD、Sentry、...
**跨端开发**：Uni-app、HarmonyOS、Android、...
...

确认无误？可增删调整。
```

#### 2.2 工作经历

从 `## 工作经历` 区域提取每段经历的公司、职位、业务背景、SARL bullet。

#### 2.3 项目经历

从 `## 项目经历` 区域提取每个项目的名称、技术栈、描述、亮点。

### 第三步：知识点映射

对每个技能项，生成对应的面试准备条目。映射逻辑：

| 技能层级 | 判断依据 | 准备深度 |
|---------|---------|---------|
| 核心技能 | 简历中列为"深度实践"，或有多个项目/经历支撑 | 深度准备：核心原理、源码级问题、项目中的设计决策、性能优化 |
| 辅助技能 | 简历中列为"项目经验"，或有 1 个项目支撑 | 标准准备：常用 API、典型场景、与核心技能的配合方式 |
| 技术视野 | 简历中列为"技术视野"，或无项目支撑 | 了解即可：核心概念、适用场景、与主力技术的差异 |

**知识点生成规则**：

对每个技能项，输出：

```
### <技能名>（<层级>）

**简历关联**：<该项目在简历中出现的位置，如"XX 项目中使用 XX 实现了 YY">

**准备要点**：
- <要点 1> — <为什么这个点在面试中重要 / 面试官会怎么问>
- <要点 2> — ...
- <要点 3> — ...

**模拟追问**：
1. <结合简历项目的具体追问>
2. <结合简历项目的具体追问>
```

**注意事项**：
- 知识点数量与层级匹配：核心技能 5-8 个要点 + 3-5 个追问，辅助技能 3-5 个要点 + 1-2 个追问，技术视野 1-2 个要点
- 追问必须结合用户的实际项目经历，不能是泛泛的面试题
- 同一知识点在多个技能中出现时，只保留一处（去重），在更核心的技能下列出

### 第四步：项目深挖准备

对每个项目经历，生成面试官可能的深挖方向：

```
### <项目名>

**一句话定位**：<简历中的一句话描述>

**面试官关心的方向**：
1. **架构决策**：<针对该项目技术选型的追问>
2. **难点攻克**：<针对该项目技术挑战的追问>
3. **量化验证**：<针对量化数据的溯源追问>
4. **如果重来**：<针对方案改进的追问>
```

### 第五步：行为面试准备

从工作经历的 SARL bullet 中提炼行为面试素材：

对每条 SARL bullet，生成一个 STARL 故事卡：

```
**故事**：<一句话概括>
**关联 bullet**：<对应的简历 bullet>
**S**：<背景> | **T**：<任务> | **A**：<方案> | **R**：<成果> | **L**：<沉淀>

**适用问题**：
- "你遇到过最大的技术挑战是什么？"
- "讲一个你推动技术决策的经历"
- ...
```

### 第六步：生成并输出

将所有内容汇总为结构化文档，按以下顺序输出：

```
# <姓名> — 面试准备清单

> 基于简历：<简历文件名>
> 生成时间：<YYYY-MM-DD>

---

## 一、技术栈总览

| 优先级 | 技能 | 层级 | 关联项目数 |
|--------|------|------|-----------|
| ⭐⭐⭐ | ... | 核心 | 3 |
| ⭐⭐   | ... | 辅助 | 1 |
| ⭐     | ... | 视野 | 0 |

---

## 二、技能知识点准备

<按优先级分组排列，每组内按关联项目数降序>

### ⭐⭐⭐ 核心技能

<第三步生成的各技能知识点>

### ⭐⭐ 辅助技能

<第三步生成的各技能知识点>

### ⭐ 技术视野

<第三步生成的各技能知识点>

---

## 三、项目深挖准备

<第四步生成的项目深挖内容>

---

## 四、行为面试故事库

<第五步生成的 STARL 故事卡>

---

## 五、考前速查

<面试前一天快速过一遍的极简清单，含：
- 核心技能关键词速查表
- 每个项目的 1 句话电梯演讲
- 3 个最强的量化成果>
```

### 第七步：验证与报告

输出并告知用户：

```
✅ 面试准备清单已生成

**输出文件**：<姓名>-面试准备清单.md

**覆盖统计**：
- 核心技能知识点：x 个（含 y 个模拟追问）
- 辅助技能知识点：x 个
- 技术视野知识点：x 个
- 项目深挖准备：x 个项目
- 行为面试故事：x 个 STARL 故事

**下一步建议**：
- 面试前 1 周：通读全部知识点 + 项目深挖
- 面试前 1 天：只看「考前速查」部分
- 面试前 30 分钟：背一遍核心量化成果 + 电梯演讲
```

## 关键规则

1. **锚定简历**：所有知识点和追问必须锚定简历中的真实技能和项目，不凭空生成
2. **分层深度**：核心技能深度准备、辅助技能标准准备、技术视野了解即可
3. **追问个性化**：模拟追问必须引用用户的真实项目背景，不列通用面试题
4. **去重合并**：同一知识点在多个技能中出现时去重，在核心技能下列出
5. **量化溯源**：对所有量化数据准备"这个数字是怎么算出来的"的回答
6. **确认再生成**：解析结果经用户确认后再进入映射步骤
7. **语言跟随用户**：用户用中文提问就用中文输出
8. **保留源文件**：不修改原始简历文件

## 边界情况

- **简历内容极少**：可能只有技能列表无项目细节 → 技能知识点正常生成，项目深挖部分标记 `<待补充>`，提示用户先完善简历
- **技能项过多（>20）**：合并相近技能，按分组层级处理，不在考前速查中列出所有项
- **无工作经历（应届生）**：跳过行为面试准备，强化项目深挖和技能知识点
- **跨行业求职**：技能映射时关注可迁移能力，弱化行业特定术语
- **用户提供了 JD**：将 JD 关键词与简历技能交叉对比，优先准备双方重叠的技能点
