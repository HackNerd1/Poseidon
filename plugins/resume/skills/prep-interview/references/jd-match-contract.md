# Worker Contract：JD 匹配分析

## 输入

- JD 文本（用户粘贴或提供链接）
- 简历解析结果（技能列表、项目列表、工作经历）

## 输出

写入 `interview/jd-match-analysis.md`（中间文件，供后续步骤引用）

## 分析流程

### 1. 提取 JD 关键词

从 JD 中识别 8-12 个高频关键词/要求，按类别分组：

| 类别 | 示例 |
|------|------|
| 硬技能 | 语言、框架、数据库、工具 |
| 领域知识 | 业务领域、行业经验 |
| 软技能 | 沟通、领导力、跨团队协作 |
| 经验要求 | 年限、项目规模、团队规模 |

### 2. 计算匹配度

对每个 JD 关键词，在简历中搜索对应证据：

| JD 要求 | 简历证据 | 匹配度 | 说明 |
|---------|---------|--------|------|
| <要求> | <简历中的技能/项目/经历> | 强匹配 / 部分匹配 / 弱匹配 / 无匹配 | <一句话说明> |

匹配度定义：
- **强匹配**：简历有此技能且列为"深度实践"，或有多个项目支撑
- **部分匹配**：简历有此技能但列为"项目经验"，或仅 1 个项目支撑
- **弱匹配**：简历有此技能但列为"技术视野"，或无项目支撑
- **无匹配**：简历中未出现此关键词

### 3. 生成应答策略

- **强匹配项** → 准备深度追问，突出量化成果，策略是"证明我有多强"
- **部分匹配项** → 准备补充说明，突出学习能力，策略是"证明我在快速成长"
- **弱匹配项** → 准备诚实应答+关联迁移，策略是"证明我可迁移"
- **无匹配项** → 准备诚实应答+学习计划，策略是"证明我学得快"

### 4. 输出优先级排序

将 JD 要求按"重要性 × 匹配度"排序，生成优先级清单：

```
优先级 1（重要且强匹配）：集中火力准备，深入原理 + 量化成果
优先级 2（重要但部分匹配）：补充准备，关联经验 + 学习轨迹
优先级 3（重要但弱/无匹配）：准备诚实应答话术
优先级 4（不太重要）：正常准备即可
```

## 质量标准

- JD 关键词提取 ≥8 个
- 每个关键词有简历中的具体证据或标注"无匹配"
- 匹配度判断有依据（基于简历中的技能层级和项目数量）
- 应答策略具体可操作，不是"好好准备"这种空话
