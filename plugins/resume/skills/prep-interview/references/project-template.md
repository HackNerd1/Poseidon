# Worker Contract：项目深挖准备文件

## 输入

- 项目名称、一句话定位、类型（个人/工作）
- 简历中的技术亮点
- JD 映射分析（如有）

## 输出

写入 `interview/projects/<project-name>.md`

## 模板

```markdown
# <项目名> — 项目深挖准备

> 一句话定位：<简历中的一句话描述>
> 类型：<个人项目 / 工作项目>
> JD 映射：<如有 JD，标注此项目可支撑 JD 中的哪些要求>

---

## 技术亮点
- <亮点 1>
- <亮点 2>
- ...

---

## 面试官关心的方向

### 1. 架构决策
**<核心问题>**
- <展开说明>

### 2. 难点攻克
**<核心问题>**
- <展开说明>

### 3. 量化验证
**<核心问题>**
- <展开说明>

### 4. 如果重来
**<核心问题>**
- <展开说明>

### 5. JD 关联追问（如有 JD）
**<结合 JD 要求的具体追问>**
- <展开说明>

---

## 模拟深挖问题
1. "<具体追问>"
2. ...
```

## 规则

- 每个方向至少 1 个问题 + 展开说明
- 量化验证部分要准备"这个数字是怎么算出来的"的回答
- 有 JD 时，新增"JD 关联追问"方向，直接对准 JD 中的关键要求
