# 简历输出模板

## 格式规范

基于《简历规范》（`references/简历规范.md`），完整简历应包含以下 6 个模块，按固定顺序排列。

### 页数控制

| 级别 | Markdown 行数（估算） |
|------|----------------------|
| 应届/初级 | 60-80 行 |
| 中级 | 80-120 行 |
| 高级/资深 | 100-150 行 |

### 通用规则

- 简体中文写作，专业术语保持英文原样
- 按"6 秒测试"原则，前半部分必须包含核心卖点
- 所有 bullet 以 `-` 开头（便于 ATS 解析）
- 禁止 emoji、图标、进度条、表格之外的复杂格式
- 链接使用标准 Markdown 格式 `[文本](URL)`

```markdown
## 个人简介

<2-3 句话，格式：年限 + 角色 + 核心技能 + 量化亮点 + 求职方向>

**示例（后端）：**
5 年后端工程师，深耕 Java 微服务与高并发系统设计，主导过日均千万级请求的支付网关架构。具备 AWS 云原生与 Kubernetes 容器化实战经验。

**示例（数据分析）：**
3 年数据分析师，擅长 SQL/Python 数据建模与 BI 看板搭建，主导过用户画像项目使留存率提升 12%。具备 A/B 实验设计与机器学习基础。

## 专业技能

**<分组名 1>：** <技能 1>、<技能 2>、<技能 3>、<技能 4>

**<分组名 3>：** <技能 1>、<技能 2>、<技能 3>、<技能 4>
**<分组名 4>：** <技能 1>、<技能 2>、<技能 3>

> 分组按岗位领域的自然分类命名（如后端可分：语言/框架、数据库/缓存、中间件/基础设施、测试/工具链；数据分析可分：分析工具、数据库、可视化、机器学习等）。

## 工作经历

### <公司名> — <职位> · <开始时间> - <结束时间>

**业务背景：** <1-2 句话，描述业务场景和挑战（SARL 的 S/T）>

- <动词> + <技术方案> + <量化成果> + <方法论沉淀>（SARL 完整）
- <动词> + <技术方案> + <量化成果> + <方法论沉淀>
- <...>

> 每段经历 2-4 条 bullet。每条聚焦一个主题，嵌入 1-2 个专业关键词。
> 动词选择：主导/设计/架构/搭建/从零构建（高级），开发/重构/优化/实现/集成（核心），维护/支持（辅助）。

**示例（后端）：**

### 某支付科技公司 — 高级后端工程师 · 2021.06 - 2024.03

**业务背景：** 负责支付网关核心系统，日均处理 500 万+ 笔交易，面临高并发场景下的稳定性与一致性挑战。

- 主导支付网关从单体迁移到微服务架构（Spring Cloud + Kafka），QPS 从 3k 提升至 15k，P99 延迟从 800ms 降至 120ms。沉淀《支付系统微服务拆分指南》，后续 2 个业务线复用
- 设计分布式事务方案（Saga + 本地消息表），解决跨行转账一致性，对账差异率从 0.3% 降至 0.01% 以下。输出《分布式事务选型决策树》，成为团队技术评审标准

**示例（数据分析）：**

### 某电商平台 — 数据分析师 · 2022.03 - 2024.01

**业务背景：** 负责用户增长与留存分析，平台月活 200 万，面临用户流失率上升的挑战。

- 构建用户分群模型（RFM + K-Means），识别高价值流失风险用户，配合运营定向干预使月留存率从 58% 提升至 70%。搭建自动化分析脚本，将周报生成从 4 小时缩短至 10 分钟

## 自我检查清单

生成后逐项确认：

- [ ] 前半部分可见：角色定位 + 最强技能栈 + 一个量化亮点
- [ ] 个人简介无套话（"性格开朗""学习能力强""团队协作好"等）
- [ ] 专业技能按领域分 3-5 组，12-20 项，第一组对齐 JD（如有）
- [ ] 无"精通""参与""各种"等减分措辞
- [ ] 每段工作经历 ≥1 个量化成果 + 方法论沉淀（L 维度）
- [ ] 每条 bullet 嵌入 1-2 个专业关键词（ATS 优化）
- [ ] 动词有力：主导/设计/开发/重构/优化（非"参与""协助"）
- [ ] 项目描述含在线链接（如有）
- [ ] 无技能进度条/百分比/图标/照片
- [ ] 无"性格""沟通能力""团队协作"等软技能罗列
- [ ] 文件命名：`姓名-目标职位.md`
- [ ] 量化数据真实（无编造），缺失的已标 `<待补充>`
```

## 模板变量说明

生成时替换以下变量：

| 变量 | 来源 |
|------|------|
| `<姓名>` | basicInfo.name |
| `<目标职位>` | basicInfo.targetRole |
| `<手机号>` / `<邮箱>` / `<GitHub URL>` / `<博客/作品集 URL>` | basicInfo 对应字段 |
| `<年限>` | basicInfo.yearsOfExperience |
| `<个人简介文本>` | summary（用户提供）或自动草拟 |
| 专业技能各组 | techStack 对象各字段 |
| 工作经历各段 | workExperience 数组 |
| 项目经历各项 | projectExperience 数组 |
| 教育背景 | education 对象 |

## JD 优化变体

当提供了 targetJD 时，在基础模板上做以下调整：

1. **技能重排**：第 1 组技能 = JD 最看重的领域
2. **个人简介微调**：突出 JD 中的核心需求
3. **bullet 关键词嵌入**：确保 JD 的 8-12 个核心关键词在 bullet 中自然出现
4. **项目排序**：与 JD 最匹配的项目放最前面
